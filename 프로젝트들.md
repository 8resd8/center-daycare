### Center-daycare

2025.12 - ing (1명)

주간보호센터 업무 효율을 향상을 위한 AI 기반 내부 업무 관리 시스템

Skills

Python, Streamlit, MySQL, Docker, GitHub Actions, Nginx

문제 상황

- 반복되는 비효율적인 상태변화기록지 작성
    - 국민건강보험공단 평가 항목인 ‘상태변화기록지’(어르신의 저번주와 이번주의 변화) 작성 및 검토의 어려움
    - 관리자 1인이 70명 어르신의 ‘상태변화기록지’를 매주 8시간 이상 대조 및 검토하며 본연의 케어 업무에 집중하지 못하는 악순환
- 매일 70명의 일일평가의 평가 확인
    - 공단 평가 항목인 ‘일일평가’ 항목의 일관성을 유지해야하는 상황
    - 직원마다 작성 방식과 내용이 달라 ‘일일평가’적합한지 검증을 반복
- 평가 항목의 문제 또는 누락 시, 기관의 평가 등급 및 운영수익 문제 발생

해결 전략

- AI 기반 텍스트 분석
    - Python과 LLM을 연동하여 70인분의 일일 기록 데이터를 분석 후, '지난주 대비 변화량'을 자동 생성하도록 설계
- 평가 가이드라인 수치화/조건화
    - 공단 평가 양식을 Prompt Engineering 및 검증 알고리즘에 구성하여 직원별 작성 내용의 일관성을 판별
- 주간상태변화 / 일일평가의 생성 및 평가
    - 검토 결과를 시각화하여 조건 미충족 데이터를 확인하고 AI 재생성

성과

- 70명의 일일 평가 검토 시간 단축: 1주일치 점검 6시간 → 30분 이내
- 상태변화기록지 평가 시간 단축: 70명의 주간 상태 비교 3시간 → 10분 이내
- 품질 평가 자동화: 국민건강보험공단 평가 양식에 맞는 ‘일일평가’ AI 평가
- 직원 성과 시각화: 개별 직원의 강점/개선점 대시보드 제공

### [Econimal](https://github.com/8resd8/Econimal)

2025.02-2025.04 (6명 / Backend 담당 / 기여도 20%)

기후 변화 교육의 수동성을 해결하기 위한 에듀테크 서비스

Skills

Java, SpringBoot, JPA, QueryDSL, MySQL, AWS EC2, Jenkins

기획 의도

- 이론 중심의 환경 교육에서 벗어나, 데이터 시각화와 퀴즈를 통해 기후 변화의 심각성을 체감할 수 있는 서비스 기획

문제 상황

- 대용량 데이터 처리 병목: 전 세계 기후 데이터 812만 건 조회 시, Full Table Scan 발생으로 인한 응답 속도 저하 및 사용자 이탈
- 부하 관리: 빈번한 통계 데이터 요청으로 인한 DB I/O 부하

해결 전략

- 데이터베이스 성능 최적화: EXPLAIN을 활용한 실행 계획 분석 후, 조회 패턴 고려한 복합 인덱스 설계
- 캐싱 도입: 변동성이 적고 요청이 잦은 데이터에 대해 Look-aside 패턴의 Redis 캐싱을 도입하여 DB 부하 분산

성과

- 조회 성능 99% 개선: 812만 건 조회 속도 10초 이상 → 100ms 미만 단축하여 사용자 경험 개선
- 서버 자원 최적화: 캐싱 도입으로 DB 접근 횟수를 줄여, 안정적인 응답 속도 유지
